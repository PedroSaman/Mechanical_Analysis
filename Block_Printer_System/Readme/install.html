ブロックプリンティングシステムのインストール

システムは，以下に示すようにいくつかのプラットフォームで動作します．

  • Windows10 ブロックモデルの生成，編集
  • Ubuntu16 ロボットの動作計画，組立計画の実行，シミュレーション
  • Denso RC7M Controller 実機の制御．シミュレーションのみなら，このプラットフォームは不要

以下では，ブロックプリンティングシステムを動作させるために必要となるソフト等のインストール手順を示します．

Windows 10

Visual Studio

Visual Studioは，C#等の言語で記述されたプログラムの開発を支援する統合開発環境(IDE)．

  1. Visual Studio 2019をインストール
  2. .NET Core SDKのバージョン2.1以上をインストール． なお，上述のVS2019のインストールが正常にできていれば，このSDKも自動的にインストールされていると思われる．

ブロックプリンティングシステム(ブロックモデル生成)のビルド

  1. program/windows/BlockModelGenerator/BlockModelGenerator.slnをダブルクリック． すると，先ほどインストールしたVisual Studio 2019が起動するはず．
  2. ウインドウ上部に3つのバーがあるはずなので，左からそれぞれ「Debug」「Any CPU」「BlockModelGenerator」を選択する．
  3. Ctrl+F5キーでビルドを開始する．ビルドが終了し，コンソール画面がでてきてなにやらメッセージが表示されれば成功．
  4. 適当なキーを押して，コンソール画面を閉じる．
  5. ウインドウ上部に3つのバーがあるはずなので，左からそれぞれ「Debug」「Any CPU」「VoxelModelConverter」を選択する．
  6. Ctrl+F5キーでビルドを開始する．ビルドが終了し，コンソール画面がでてきてなにやらメッセージが表示されれば成功．
  7. 適当なキーを押して，コンソール画面を閉じる．

Unity

Unityは，3Dグラフィックを扱うゲーム等のソフト開発を支援するツール．

  1. Unityをインストール．
  2. program/windows/BlockPainter-UnityProject.zipを展開．
  3. できたフォルダ内のAssets/scene1.unityをダブルクリック． すると，Unityが開く．

Ubuntu 16

ROS

  1. ROS (Robot Operating System) のKineticバージョンを，ROSに沿った手順でインストールする． なお，当該ページの「1.4 Installation」ではインストールするパッケージの選択肢があるが，「Desktop-Full Install: (Recommended)」をインストールすればよい．
  2. moveit!のパッケージをインストール．
    $ sudo apt install ros-kinetic-moveit-*

ブロックプリンティングシステム(ロボット動作計画)のビルド

  1. 引継ぎ資料のprogram/ubuntuの中身をホームディレクトリ直下に移動する． ホームディレクトリ直下に「package_ws」「block_printer_ws」「denso_ws」の3種類のディレクトリがあれば良いことになる．
  2. 端末を開く．
  3. 以下のコマンドでビルド開始．
    $ cd ~/package_ws & catkin_make
    $ cd ~/block_printer_ws & catkin_make
    $ cd ~/denso_ws & catkin_make
  4. 最後に，端末を起動するたびに自動的にワークスペース内のパッケージ情報を読み込むように設定．
    $ source /opt/ros/kinetic/setup.bash $ echo "source ~/package_ws/devel/setup.bash" >> ~/.bashrc
    $ echo "source ~/block_printer_ws/devel/setup.bash" >> ~/.bashrc
    $ echo "source ~/denso_ws/devel/setup.bash" >> ~/.bashrc
  5. 端末を閉じる．

Denso RC7M Controller

この項目は，ブロックプリンティングのシミュレーションのみを行う場合は不要．

  1. 引継ぎ資料のprogram/DensoRC7MControllerの中身をすべてRC7Mコントローラに送信する．
  2. ティーチングペンダントでプログラムのコンパイル．

なお，他の人がコントローラの中身を削除していなければ，本研究で利用したプログラムがそのまま保存されているはずです． その場合，この操作は不要です．
