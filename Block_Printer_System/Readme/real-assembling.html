実機(Denso VP6242)によるブロックプリンティング

以下では，デンソーウェーブ製6軸ロボットVP-6242Gと，ロボットコントローラRC7Mを利用したブロックプリンティングの手順を示す．

ロボットコントローラRC7Mへプログラム転送

この項目は，RC7M内のブロックプリンティング用プログラムが削除されていなければ行わなくて良い．

  1. 「program_for_rc7m.zip」の中身を展開． すると，ロボットコントローラで処理されるpac形式のロボットプログラムができる．
  2. 展開後のフォルダにあるpacファイルを，RC7Mコントローラのxdntaiyoesgフォルダに転送する．
  3. ロボットコントローラ上でプログラムをビルドする．ビルドはコントローラに付いているティーチングペンダントから行える．

ブロックのピック・プレイス位置のキャリブレーション

このキャリブレーションは，かなり丁寧に行わないと組立に失敗することになる． 組立に失敗するとき，ブロックがロボットに潰されたり，ブロックがはじけ飛んで自分に向かってくることがあった．危険だ．

  1. RC7コントローラの電源を入れる．
  2. コントローラの自動イネーブルがOFF，手動モードになっていることを確認する．
  3. ティーチングペンダントからサーボの電源を入れる．
  4. Ubuntuの端末を4つ開き，それぞれに以下のコマンド．これは以下の順で行う必要がある．
    $ roscore
    $ roslaunch denso_bcap_controller controller.launch readonly:=true
    $ roslaunch denso_bcap_controller interface.launch
    $ roslaunch denso_bcap_controller moveit.launch 最後のコマンドまで実行すると，rvizというGUIソフトが起動する．
  5. エクスプローラでdenso_ws/vp6242/vp6242_description/config/pick.yamlと denso_ws/vp6242/vp6242_description/config/place.yamlを開く．
  6. ティーチングペンダントからロボットを操作して，各ブロックのピック位置まで移動．
  7. rvizに表示されるロボットのリンクのうち，「peak-tool」リンクの位置~(x,y,z)と姿勢~(x,y,z,w)を， pick.yamlの「position」の行と「orientation」の行に記録する．
  8. ピック位置のキャリブレーションが終わったら，プレイス位置のキャリブレーションを行う．
  9. キャリブレーションが終わったら，ロボットのサーボ電源を切る．
  10. 開いている4つの端末上でctrl+Cを押して，システムを終了させる．

自動組立

  1. グリッパのケーブルをゆるめに束ねておく． これは，ケーブルが予想外の動きをしてブロック等と干渉するのを防ぐため． また，きつく束ねてしまうと，ロボットが大きく動いたときにケーブルが引っ張られて破壊される恐れがある．
  2. RC7コントローラの電源を入れる．
  3. コントローラの自動イネーブルがON，手動モードになっていることを確認する．
  4. ティーチングペンダントからサーボの電源を入れる．
  5. コントローラを自動モードにする．
  6. Ubuntuの端末を5つ開き，順に以下のコマンド．これは以下の順で行う必要がある．
    $ roscore
    $ roslaunch denso_bcap_controller controller.launch readonly:=true
    $ roslaunch denso_bcap_controller interface.launch
    $ roslaunch denso_bcap_controller moveit.launch
    $ roslaunch vp6242_description block_printer_commander.launch
    $ block_model_filename:=組立計画ファイルへのフルパス block_model_type:='kohama2019'

自動組立後に

  1. デスクトップに，動作計画に要した時間と動作の実行に要した時間を記述したcsvファイルがそれぞれ出力される．
  2. 組み立てられたサブアセンブリ同士を合体させ，サポートブロックを取り除くと完成品となる．

ほかのロボットを使いたいとき

以下のインターフェースを作ってやればよい．

  • 実機の現在の関節角を，MoveIt!側に常に送信(ROS publisher)してやるプログラム．
