ブロックモデルのつくりかた

以下では，Windows側のブロックプリンティングシステムで，ブロックの組立計画を生成する手順を紹介します．

ボクセルモデルをつくろう

  1. 適当な3Dモデルを用意する．3Dモデルのファイル形式はSTLを推奨．
  2. ブラウザでdrububu.comにアクセスする．
  3. 「open file」ボタンをクリックし，3Dモデルを選択する．
  4. 「open file」ボタンの下にある「.json」トグルボタンを有効にする．
  5. 「open file」ボタンの上や右側のバーを動かして，ボクセルモデルのサイズや充填率を好きなように設定する．
  6. 「save as json」ボタンを押して，json形式で記述されたボクセルモデルをダウンロード．

ブロックモデルをつくろう

例として，「block-model-source/vovel json」にあるボクセルモデル「bunny674.json」を使う．

  1. コマンドプロンプトを開く．
  2. コマンドプロンプト上で，「program/windows/BlockModelGenerator」フォルダへ移動．
  3. 以下のコマンド．
    > dotnet run --project VoxelModelConverter PATH
    (「Path」のところには，bunny674.jsonの相対パスまたは絶対パスを入力する)．
  4. 長々としたメッセージのあと，「block-model-source/BlockModel」フォルダに，「bunny674.txt」というファイルが生成されるはず．これがブロックモデル．

ブロックモデルに色を付けよう

  1. 「program/windows/BlockPainter-UnityExecutable/BlockPainter-win.exe」を起動．
  2. ウインドウサイズと画質を適当に選んで，「Play!」ボタンをおす．すると，Unityで製作したプログラムが起動するはず．
  3. ウインドウ左端の「Load Model」ボタンを押すと，ファイルダイアログが表示される．
  4. ダイアログ内で先ほど生成した「bunny674.txt」を選択する．
  5. すると，生成したブロックモデルが表示される．
  6. ウインドウ左の色一覧から，「Blue」を選択し，続けて「Apply Color」を押す．
  7. キーボードのZキーを押し，マウスを左クリックした状態で，マウスポインタでブロック上をなぞってみる． すると，なぞったブロックがBlue色になる．
  8. このようにして，好きな色をブロックに割り当てていく．なお，以下の操作を知っていると，スムーズに色付けができる．
    • マウス左クリック+マウスポインタを動かす…カメラ回転
    • マウス右クリック+マウスポインタを動かす…カメラ平行移動
    • マウスホイールを動かす…カメラのズームイン/ズームアウト
    • キーボードのXキーを押し，マウスを左クリックした状態で，マウスポインタでブロック上をなぞる…なぞったブロックを消す． ブロックモデルに余計なブロックがある場合に使ってみよう．
    • Ctrl+Z ブロックモデルを，ひとつ前の状態に戻す．
    • キーボードのDownArrow/UpArrow…ブロックモデルのうち，上段ブロックを非表示/表示する． ブロックモデル内部のブロックに色を割り当てたい場合に使おう．
  9. 色付けが終わったら，ウインドウ左側の「Save Model」ボタンをクリック．
  10. ファイルパスを指定して「Save」を押すと，色付きブロックモデルファイルが生成される．

ブロックの組立計画をつくろう

ブロックプリンティングを行うには，ブロックの適切な組立順を計画してやる必要がある．

  1. コマンドプロンプトを起動．
  2. コマンドプロンプト上で，「program/windows/BlockModelGenerator」フォルダへ移動．
  3. 以下のコマンド．
    > dotnet run --project BlockModelGenerator PATH
    ただし，「PATH」の部分には，先ほど生成した色付きブロックモデルの相対パスまたは絶対パスを入力すること．
  4. とても長いカラフルな出力とそれなりの時間のあと，以下のような形式の組立順候補がいくつか表示される．

この候補の番号/候補数
サブアセンブリの数
総ブロック数 (各サブアセンブリのブロック数)
総サポートブロック数 (各サブアセンブリ中のサポートブロック数)

  1. それぞれの組立順候補について，適当な評価値を整数で入力してやる． サブアセンブリが少ないものや，サポートブロックが少ないものなど，より組み立てやすそうな候補に高い評価値を与えよう．
  2. 最高の評価を与えた候補だけが保存される．なお，このとき各サイズ，各色のブロックが何個必要かが画面上に出力される．

なお，このときカレントディレクトリに生成される「AssemblyPlanSetting.json」というファイルの中身を編集すると，組立計画時の挙動を変更することができる． 各項目の意味は下記のとおり．

  • AssemblyAreaOrigin…ブロックモデルが配置される基準となる位置．
  • AssemblyAreaSize…ブロック組立パッドのサイズ．実機での組立環境に応じて，適宜変更してやる．
  • AvailableSupportBlockSizes…利用可能なサポートブロックのブロックサイズを配列で定義してやる．
  • MaxSandwichedSupportCount…モデルを構成するブロック同士に挟まれたサポートブロックの上限数． 通常，挟まれたサポートブロックを取り除くのは面倒なので，このパラメータは低めに設定してやることが望ましい．
  • MaxSubassemblyCount…最大サブアセンブリ数． 通常，サブアセンブリがたくさんあると最後に合体させるのが面倒なので，このパラメータは低めが望ましい．
  • MinSubassemblyLayerCount…各サブアセンブリがもつべき最小の段数． 段数が少ない (高さが低い)サブアセンブリは，組立パッドから外すのが大変なので，このパラメータは高めが望ましい．

(Optional)組立計画を確認してみよう

  1. 「BlockPainter-UnityExecutable/BlockPainter-win.exe」を起動．
  2. 先ほどの要領で，組立計画ファイル(block-model-source/plan下のcsvファイル)を開く．

注意

3D CADモデル，ボクセルモデル，ブロックモデル，色付きブロックモデル，組立計画ファイルはそれぞれ 「mesh」「voxel json」「BlockModel」「ColorModel」「plan」というフォルダに保存しておくことを推奨． また，これらのフォルダは同一のフォルダに存在することが推奨される (引継ぎ資料の「block-model-source/」内と同様のフォルダ構造であった方が良いということ)． これは，たしか，プログラムにこのフォルダ構造を前提とした実装があったかもしれないため…．
